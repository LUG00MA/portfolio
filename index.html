Cahier des charges

💡
Technologies en termes simples
  • Architecture MVC — organisation du code en 3 parties séparées : la base de données (Modèle), les pages affichées (Vue) et la logique du site (Contrôleur). Cela rend le code plus propre et maintenable.
  • CRUD — les 4 actions de base sur les données : Créer, Lire, Modifier et Supprimer (Create / Read / Update / Delete).
  • Sessions PHP — mécanisme qui garde l’utilisateur connecté lorsqu’il navigue de page en page.
🎯

Objectif

Créer une application de gestion en architecture MVC avec PHP et MySQL.

👥

Technologies en termes simples
  • Leaflet.js — bibliothèque JavaScript gratuite qui permet d’afficher des cartes interactives dans un navigateur (similaire à Google Maps, mais intégrée directement dans le site).
  • AJAX — technique permettant de mettre à jour une partie de la page sans la recharger entièrement. C’est ce qui donne l’effet « temps réel ».
  • Architecture MVC — organisation du code en Modèle (base de données), Vue (affichage) et Contrôleur (logique), pour un projet propre et évolutif.
  • Back-office admin — interface réservée aux administrateurs pour gérer les vélos, les stations et les utilisateurs (invisible du public).

💡
Technologies en termes simples
  • C# WinForms — langage Microsoft (C Sharp) avec un framework pour créer des applications bureau Windows avec des fenêtres, boutons et formulaires (contrairement à un site web).
  • JSON — format de fichier texte pour stocker des données structurées (comme un profil utilisateur ou un post), lisible par les humains et les machines.
  • CRUD sur les profils — l’application permet de Créer, afficher (Lire), modifier (Update) et Supprimer des profils et des publications.
🎯

Objectif principal

Développer un réseau social fictif en C# WinForms simulant Facebook : profils, posts, likes, fil d’actualité.

📄

Stockage et lecture des données utilisateurs (profils, posts, images) via des fichiers JSON — un format texte structuré lisible par les machines.

🌔

Mode sombre

👥

Gestion des profils

CRUD complet. Recherche par pseudo et affichage détaillé (bio, localisation, parcours).
